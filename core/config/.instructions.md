# ⚙️ core/config/ - 配置管理準則

本模組負責全域配置與環境變數的解析與驗證，基於 Pydantic Settings 構建。

---

## 🔧 擴充規範

當你需要新增配置參數時，請遵循以下步驟：

1. **參數定義**：在 `settings.py` 的 `Settings` 類別中新增屬性，並提供合理的預設值。
2. **類型標註**：必須標註明確的類型（如 `str`, `int`, `List[str]`）。
3. **環境變數對應**：屬性名稱即為環境變數名稱（不分大小寫）。
4. **驗證邏輯**：使用 Pydantic 的 `Field` 或 `validator` 進行資料驗證。

---

## 📋 數據模型規範

若需定義跨模組使用的數據結構（如 `SearchResult`）：
- 檔案位置：應放在 `core/config/models.py`。
- 技術選型：一律繼承 `pydantic.BaseModel`。
- 註解：每個欄位都必須有 `Field(..., description="...")`，這對 Agent 理解數據至關重要。

---

## 🚀 使用與匯出

- **唯一實例**：系統應只使用 `from core.config import settings` 取得全域實例。
- **重新導出**：所有模型應在 `__init__.py` 中導出，方便其他模組直接以相容路徑導入。

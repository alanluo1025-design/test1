# 關聯式資料庫模組

負責 SQLAlchemy 連線管理、ORM 模型定義與資料庫遷移腳本。

---

## 職責範疇

| 檔案/目錄 | 職責 |
|---------|------|
| `database.py` | Engine 建立、Session 工廠、`init_db()` 建表、`get_db()` 依賴注入 |
| `models.py` | 所有 SQLAlchemy ORM 模型定義（對應 `資料庫結構設計.md`） |
| `migrations/` | Alembic 遷移腳本（結構變更時新增，不手動修改資料庫） |

---

## 使用方式

```python
# FastAPI 依賴注入取得 Session
from core.db import get_db

@app.get("/example")
def example(db: Session = Depends(get_db)):
    ...

# 匯入 ORM 模型
from core.db.models import User, NewsArticle, Collection
```

`init_db()` 只應在應用程式啟動入口（`main.py` 或 FastAPI `lifespan`）呼叫一次，不得在模組層級自動執行。

---

## 擴充規範

- 新增 ORM 模型時，必須同步更新 `__init__.py` 的 `__all__`
- 模型命名使用 PascalCase 單數名詞（`NewsArticle`，非 `NewsArticles`）
- 所有 `Mapped` 欄位必須標注類型，可為 null 一律用 `Mapped[Optional[...]]`
- 欄位加 Index 時統一寫在 `__table_args__` 中，不散落在欄位定義裡

---

## 切換 PostgreSQL

只需修改 `.env` 的 `DATABASE_URL`，模型定義無需改動：

```
# 開發（SQLite）
DATABASE_URL=sqlite:///./data/app.db

# 正式（PostgreSQL）
DATABASE_URL=postgresql://user:pass@host:5432/dbname
```

---

## ⚠️ 禁忌

- ❌ 不得在 `models.py` 內撰寫任何商業邏輯或資料驗證以外的函數
- ❌ 不得在 `core/db/` 以外直接使用 `SessionLocal()` 建立 Session，一律透過 `get_db()`
- ❌ 不得在模組 import 時自動執行 `init_db()`，建表只能由啟動入口觸發
- ❌ 不得直接操作 `data/app.db` 檔案，所有讀寫透過 SQLAlchemy ORM

# 📝 core/logging/ - 日誌系統規範

負責全域日誌的生命週期管理與格式標準。

---

## 🚀 使用方式

每個模組取得 logger 的唯一方式：

```python
from core.logging import get_logger

logger = get_logger(__name__)   # __name__ 讓日誌路徑可追溯到原始碼
```

不允許直接使用 `import logging; logging.getLogger()`，所有日誌配置應集中在 `logger.py` 管理。

---

## 🏗️ 設計準則

1. **模組化記錄器**：必須使用 `get_logger(__name__)`，確保日誌路徑能追溯到原始碼位置。
2. **單一出口**：所有日誌配置封裝在 `logger.py`，外部模組不應手動配置系統 `logging`。

---

## 📋 日誌級別選擇標準

- `DEBUG`：開發調試用的詳細資訊（如爬蟲抓到的原始內容）。
- `INFO`：業務流程的關鍵節點（如「Agent 開始執行」、「檢索成功」）。
- `WARNING`：非致命異常，系統仍可運行。
- `ERROR`：影響功能運行的致命錯誤。

---

## 🎨 格式與風格

- **語系**：日誌訊息必須使用**繁體中文**。
- **Emoji**：關鍵日誌應搭配 Emoji 以提升可讀性（✅ 成功, ❌ 失敗, ⚠️ 警告, 🔍 查詢, 💾 存儲, 📥 載入）。

```python
logger.info("✅ 資料已成功寫入")
logger.warning("⚠️ 快取已過期，重新抓取")
logger.error("❌ 連線失敗，請檢查網路")
```

---

## ⚠️ 禁忌

- ❌ 不得在日誌中印出 API Key 或密碼等敏感資訊
- ❌ 不得在 `logger.py` 以外的地方呼叫 `logging.basicConfig()`

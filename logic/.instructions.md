# 🧠 logic/ - Agent 決策層規範

本模組負責 Agent 定義、LLM 工廠、提示詞管理、workflow 編排。

---

## 🎯 各檔案職責與禁忌

| 檔案 | 職責 | ❌ 禁忌 |
|:---|:---|:---|
| `agents.py` | `create_agent_node()`、LLM 工具綁定、記憶體截斷邏輯 | 工具實現、配置管理、提示詞 |
| `llm_factory.py` | 多模型工廠 `get_llm()`、環境變數驗證、模型初始化 | 工具相關代碼、提示詞、配置存儲 |
| `workflow.py` | LangGraph StateGraph、邊界定義、條件邊界、節點連接 | 節點內部邏輯、工具定義 |
| `prompts.py` | 系統提示詞字串、動態提示詞組合函數 | 配置讀取、工具邏輯、決策判斷 |

---

## 🤖 agents.py 關鍵規範

1. **記憶體截斷必須存在**：`agent_node` 執行前必須截斷過長的 `messages`，保護 Context Window，閾值參考 `settings.rag_threshold`。
2. **工具綁定用 `bind_tools()`**：不要手動呼叫工具，讓 LangGraph 的 `ToolNode` 自動路由。
3. **函數回傳格式固定**：`agent_node` 永遠回傳 `{"messages": [response]}`。

---

## 🏭 llm_factory.py 關鍵規範

1. **新增模型三步驟**：① 在 `get_llm()` 加 `elif` 分支 → ② 在 `core/config/settings.py` 加對應金鑰 → ③ 在 `.env` 加範例說明。
2. **金鑰未設定一律 `raise ValueError`**，不允許靜默失敗。
3. **日誌格式統一**：`logger.info(f"🟢 [系統] 當前載入模型：{provider} ({model_name})")`

---

## 🌊 workflow.py 關鍵規範

1. **圖構建封裝在 `_build_graph()`** 私有方法內，對外只暴露 `chat()` 一個公開介面。
2. **節點順序固定**：`START → agent → tools → agent`（條件邊界由 `tools_condition` 判斷）。
3. **`chat()` 必須處理 `list[dict]` 格式**：部分模型（如 Gemini）回傳 content 為 list，需統一轉為純字串後回傳。

---

## 📝 prompts.py 關鍵規範

1. **提示詞必須包含五個部分**：身份定義 → 當前日期 → 工作流步驟 → 工具路由指南 → 輸出格式要求。
2. **動態內容用參數注入**，不可在提示詞中硬編碼日期或使用者資訊。
3. **私有 helper 函數用 `_` 開頭**，避免被外部誤用。

---

## ⚠️ logic/ 不應該做的事

- ❌ 直接呼叫工具函數（工具由 LangGraph `ToolNode` 自動呼叫）
- ❌ 直接進行 I/O 操作（透過 `core/` 的模組）
- ❌ 硬編碼模型名稱或 API 金鑰（透過 `core.config.settings`）
- ❌ 實現向量搜尋或資料庫寫入（透過 `core.rag` 或 `core.db`）

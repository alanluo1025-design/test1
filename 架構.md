## 目錄詳細說明

### 1. 執行入口 (Entry Points)
| 檔案名稱 | 功能描述 |
| :--- | :--- |
| `discord_bot.py` | Discord 機器人啟動與監聽主程式 |
| `main.py` | 命令列 (CLI) 終端機對話互動主程式 |

### 2. 系統核心 (`core/`)
負責系統基礎建設、設定與底層服務。

| 目錄 | 功能描述 |
| :--- | :--- |
| `config/` | 配置管理 (Pydantic Settings、環境變數解析) |
| `logging/` | 日誌系統 (全域記錄器工廠、日誌配置) |
| `db/` | **關聯式資料庫管理** (SQLAlchemy Engine、Session、ORM 模型、Alembic 遷移) |
| `rag/` | 向量搜索與 RAG (ChromaDB 連線、Embedding、重排序) |
| `utils/` | 通用工具 (內容處理 `processor.py`、調試工具 `debug.py`) |
| `tools/` | **AI Agent 技能與工具目錄** (PDF、RAG、爬蟲、搜尋) |
| `ai_agent_db/` | ChromaDB 向量資料庫實體檔存放目錄（不入 git） |

### 3. 資料儲存層

| 路徑 | 類型 | 說明 |
| :--- | :--- | :--- |
| `data/app.db` | SQLite（開發） | 關聯式 DB 實體檔，正式環境改為 PostgreSQL |
| `core/ai_agent_db/` | ChromaDB | 向量資料庫實體檔，儲存使用者偏好與典藏文章向量 |

> 切換 PostgreSQL：修改 `.env` 的 `DATABASE_URL=postgresql://...`，程式碼無需變動。

### 4. 邏輯與決策 (`logic/`)
定義代理人的決策流程與模型調度。

| 檔案/目錄 | 功能描述 |
| :--- | :--- |
| `agents.py` | Agent 大腦節點定義 (對話紀錄截斷、提示詞載入) |
| `llm_factory.py` | 模型生成工廠 (負責建立 Gemini, Ollama, NVIDIA 等實例) |
| `prompts.py` | 系統提示詞庫 (行為準則、人設定義) |
| `workflow.py` | LangGraph 流程編排 (節點、邊界、工具路由) |

## 開發與測試

- **`test/`**: 開發者測試腳本（模型連線、RAG 功能測試）
- **`debug_logs/`**: 運行時自動生成的除錯日誌，記錄爬蟲抓取的原始文本
- **`.env`**: 存放 API Keys 等機密資訊

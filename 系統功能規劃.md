# 專題系統功能規劃 - AI 技術洞察平台

## 一般功能

1. **第三方登入**：為了記錄每位用戶的個人資料，必須使用帳戶才能登入我們的系統。這些帳號可以是 Discord、Line、Facebook、Google 等較為人知或較常使用的帳戶。

2. **客製化AI新聞主題**：第一次登入系統時，系統會先詢問每位使用者喜歡的新聞主題。系統會以勾選標籤的形式供使用者做選擇，系統就會針對此主題的AI新聞作收集，再加上抓取新聞Agent自行判斷的近期火紅的AI新聞。爾後還能修改欲搜尋的主題。

3. **瀏覽每日新聞看板**：將系統每天蒐集到的新聞內容先透過語言模型做修改，接著呈現在介面上供使用者觀看。使用者起初只能看到每篇新聞的標題和簡短摘要。若想進一步了解內容，用戶能點擊系統提供的連結，前往至另一頁面觀看完整訊息。

4. **生成摘要**：透過語言模型將新聞內容濃縮成 100-200 字的簡述。

5. **新聞語音檔生成**：將新聞內容文字檔轉成語音檔，提供給那些處於無法閱讀的使用者一個獲取新知的替代方案。提供線上即播放或是語音檔下載的連結。

6. **討論版/留言板**：為了符合社群的概念，此系統結合大眾討論的理念。每位用戶能在系統內建立討論區，用戶還可回覆用戶留言，分享彼此對於某些新聞內容的看法。

7. **社群平台資訊推送**：系統的訊息或通知透過第三方通訊平台或是以電子郵件形式發送給使用者，使用者能在熟悉的介面獲取部分消息。

8. **典藏**：可選擇將自己喜歡的新聞做典藏，透過RAG將語言模型修改後的新聞原文暫時存取在個人資料頁面中，便於之後拿出來反覆觀看或引用。
   - 由於資料庫空間的限制，須限制使用者暫存的數量（50則）和定時刪除資料
   - 刪除資料分為三種：定時清理、滿50則清理、使用者手動清理

## Agent

1. **個人化新聞推播Agent**：系統起初會分析客製化新聞主題的結果作為推播的依據。等到使用者使用平台一段時間後，他會自動分析用戶的暫存的新聞，選取用戶可能喜歡的新聞，透過第三方通訊平台發送新聞消息。用戶還可自訂發送的頻率或是新聞的數量。

2. **主題引導式搜索Agent**：Agent 先與使用者進行幾輪對話，再根據對話的結果建立專屬於每位用戶的情報蒐集員。Agent 會調用搜尋工具尋找使用者想要的特定新聞。和一般模式的不同在於客製化的搜尋。

3. **提問/觀點/批評/回覆式Agent**：自行判斷討論版內容，並且產出針對此版主的內容的問題或觀點，帶動其他使用者思考的方向與動力。自行判斷且針對討論板下用戶的留言做批評或回覆，增加互動的多元性。

4. **抓取AI新聞Agent**：自動判斷日前AI領域熱門新聞網站，並使用預先設定好的網站名單。透過tavily工具鎖定這些網站，再將新聞資料回傳給系統。

5. **「數位雙生」讀書分身**：這是一個高度個人化的 Agent，它代表的是「你未來的知識儲備」。

### 「數位雙生」讀書分身功能

**功能**：這個 Agent 會學習使用者的閱讀品味與技術程度。當它抓到新聞時，它會先進行「模擬閱讀」

**價值**：它幫使用者過濾掉已知資訊，只餵養「純粹的新知增量」

### 「數位雙生」Agent的流程

#### 第一步：構建Agent的知識庫

系統會持續追蹤並更新使用者的歷史紀錄：

- **來源**：暫存過的新聞、參與討論過的觀點、搜尋過的主題
- **儲存**：會將這些資料傳送給數位雙生 Agent，轉換成向量儲存在向量資料庫中

#### 第二步：模擬閱讀與比對

1. 它會先讀完全文
2. 它會把文章拆解成多個知識點（如：向量資料庫、RAG、動態索引）
3. 它會去使用者的歷史紀錄比對，發現使用者早就懂 RAG 了，但對「動態索引」還很陌生

#### 第三步：精準引導閱讀

最後產出一段高度個人化的對話：

「使用者，這篇新聞我讀過了。前面講的硬體規格跟你上週看的那篇一樣（略過），但它提到的『冷熱資料自動分層』剛好可以解決你正在做的專題中的延遲問題，建議直接看第四段。」
